# Deep Dict

Deep Dict is a compact English-Chinese dictionary for macOS Dictionary.app and
three-finger Lookup. It is designed as a reading aid: short Chinese glosses,
common senses first, and only lightweight domain labels where they help.

## Current Release

The current local build contains:

- 91,813 included entries
- 8,870 model-excluded candidate words
- 0 generation error records in the final run

Compiled release assets are prepared under `release/`.

## Repository Layout

- `prompts/entry-generation-system.md`: prompt used to generate entry JSON.
- `scripts/generate_entries.py`: parallel DeepSeek-compatible entry generator.
- `scripts/build_apple_dictionary_source.py`: converts entry JSON into Apple
  Dictionary Development Kit source.
- `scripts/watch_and_build_deep_dict.py`: optional watcher that builds after a
  generation run finishes.
- `scripts/build_*_wordlist.py`: word-list discovery utilities.
- `data/generation-targets/merged-clean/words.csv`: production candidate word
  list used for the latest full run.
- `data/final-wordlists/included-words.csv`: included headwords from the latest
  final run.
- `docs/lexicon-source-plan.md`: notes on source data and workflow decisions.

Generated logs and build outputs are not tracked by default. The current release
does include a compressed generated-entry archive under `release/` so the
dictionary data can be inspected without rerunning the model.

## Install Prebuilt Dictionary

Use the zip in `release/DeepDict.dictionary.zip`.

After extracting it, copy `DeepDict.dictionary` to:

```sh
~/Library/Dictionaries/
```

Then open Dictionary.app and enable `Deep Dict` in settings.

## Regenerate Entries

Create a Python environment and install dependencies:

```sh
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Set an API key for a DeepSeek-compatible chat endpoint:

```sh
export DEEPSEEK_API_KEY="..."
```

Run generation:

```sh
python scripts/generate_entries.py --workers 64
```

The generator writes one JSON file per word under `outputs/entries-merged-clean`
and validation errors under `outputs/errors-merged-clean`.

## Build macOS Dictionary

Apple's Dictionary Development Kit is required to compile the `.dictionary`
bundle. The DDK is not bundled in this repository.

Generate DDK source files:

```sh
python scripts/build_apple_dictionary_source.py \
  --entries-dir outputs/entries-merged-clean \
  --output-dir build/deep-dict \
  --dict-name "Deep Dict" \
  --package-name DeepDict \
  --display-name "Deep Dict" \
  --bundle-id org.deepdict.dictionary.en-zh \
  --manufacturer "Deep Dict Project" \
  --xml-name DeepDict.xml \
  --clean
```

Compile with the DDK:

```sh
make -C build/deep-dict DICT_BUILD_TOOL_DIR="/path/to/Dictionary Development Kit"
```

Install locally:

```sh
make -C build/deep-dict install
```

## Data Notes

The word list was built from a cleaned broad English candidate pool plus corpus
supplements from technical and discussion sources. Entries were generated by an
LLM and should be treated as practical reading-aid data, not an authoritative
professional dictionary.

Before publishing under an open-source license, review third-party source-list
redistribution constraints and choose an explicit license for code and generated
dictionary data.
